# DEX
### Data & Electrodermography eXperiment

A live GSR read detector and feedback loop for building a labeled EDA dataset.  

---

## What this is

DEX watches a live 20Hz GSR stream, detects candidate phasic events, classifies them using a random forest trained on annotated data, and asks you to confirm or reject each call. Every 👍👎 becomes a training example. The model gets smarter every time you run `train.py`.

```
ESP32 → BLE/WS → browser (index.html)
                      ↓  20Hz frames
               dex.py (FastAPI/WS)
                      ↓  call: "read" | "other"
               browser → 👍 👎
                      ↓  feedback
               feedback.jsonl
                      ↓  next training run
               dataset.csv → dex.joblib
```

---

## Hardware

- **XIAO ESP32S3** running `e-meter.ino`
- **Grove GSR sensor** (finger electrodes)
- Firmware outputs JSON at 20Hz via BLE, WebSocket (port 5002), and SD card
- Frame format: `{"t": ms, "smooth_uS": float, "delta": float, "velocity": float}`

## Setup

### On a Linode (or any Linux box)

```bash
git clone <your-repo> dex
cd dex

python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

Point a domain at your Linode IP and set up SSL (certbot + nginx recommended).  
DEX expects to run behind a reverse proxy at `wss://api.yourdomain.com/stream`.

### Minimal nginx config

```nginx
location /stream {
    proxy_pass         http://127.0.0.1:8000/stream;
    proxy_http_version 1.1;
    proxy_set_header   Upgrade $http_upgrade;
    proxy_set_header   Connection "upgrade";
}

location /health {
    proxy_pass http://127.0.0.1:8000/health;
}
```

## Workflow

### After each session

1. Export annotations from the editor (`export jsonl` button → unzip)
2. Drop the JSONL files into `annotations/`
3. Run:

```bash
python3 collate.py   # merges all annotations → dataset.csv
python3 train.py     # retrains forest → dex.joblib
```

That's it. The running server picks up the new model automatically on next load  
(restart `serve.py` to reload mid-session).

### Start the server

```bash
uvicorn serve:app --host 0.0.0.0 --port 8000
```

For production, run under `systemd` or `screen`/`tmux`:

```bash
# simple tmux approach
tmux new -s dex
uvicorn serve:app --host 0.0.0.0 --port 8000
# Ctrl+B D to detach
```

### Check it's running

```bash
curl https://api.yourdomain.com/health
# {"status": "ok", "model": {"n_samples": 29, ...}}
```

## Connecting the browser

In `index.html`, add one WebSocket connection after BLE connects:

```javascript
const dex = new WebSocket("wss://api.yourdomain.com/stream");

// forward frames to DEX
dex.onopen = () => console.log("[dex] connected");

// render calls from DEX
dex.onmessage = e => {
    const msg = JSON.parse(e.data);
    if (msg.type === "call") {
        showCall(msg);   // your UI — flash, text, sound, whatever
    }
};

// in your existing handleFrame():
function handleFrame(f) {
    // ... existing needle/polygram code ...
    if (dex.readyState === WebSocket.OPEN) dex.send(JSON.stringify(f));
}

// feedback buttons
function sendFeedback(eventId, label) {
    dex.send(JSON.stringify({ type: "feedback", event_id: eventId, label }));
}
```

## The feedback loop

DEX emits:
```json
{"type": "call", "event_id": "a3f9c1b2", "label": "read", "confidence": 0.84, "features": {...}}
```

Your UI shows 👍 👎. On click:
```json
{"type": "feedback", "event_id": "a3f9c1b2", "label": true}
```

Every confirmed or rejected call is appended to `feedback.jsonl`.  
On training day, fold `feedback.jsonl` into your annotations and retrain.

---

## Training day

You don't need a GPU. On a basic Linode or your laptop:

```bash
python3 collate.py
python3 train.py
```

Training 29 samples takes under a second. Even 10,000 samples will train in minutes on CPU with scikit-learn. A GPU only becomes relevant if you move to raw waveform input (1D-CNN) — not yet.

**Recommended cadence:** retrain whenever you have a meaningful batch of new feedback, roughly after every few sessions early on. The LOO accuracy printed by `train.py` tells you if the model is improving.

---

## What the model sees

Four features extracted from each candidate event window:

| Feature | What it captures |
|---|---|
| `amplitude` | size of the phasic response in µS |
| `attack_s` | onset speed — artifacts are near-instantaneous |
| `release_s` | decay time — real reads release slowly |
| `baseline_uS` | tonic level at event time — session context |

Accuracy on 29 samples: **69% LOO** — meaningful, not final.  
Amplitude and release_s are the strongest discriminators.

## Detector tuning

In `serve.py`, three constants control sensitivity:

```python
DELTA_DROP_THRESH   = 0.8   # minimum delta drop to fire a candidate
MIN_DURATION_FRAMES = 8     # ~400ms minimum event length
COOLDOWN_FRAMES     = 40    # ~2s lockout after each candidate
```

If DEX is firing too often (noisy), raise `DELTA_DROP_THRESH`.  
If it's missing reads, lower it. Tune based on your feedback.jsonl patterns.
